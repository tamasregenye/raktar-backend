6. Rendezés (ORDER BY)
    - A lekérdezések eredménye alapértelmezetten abban a sorrendben jelenik meg, ahogy az adatok az adatbázisba bekerültek.

    - ORDER BY
    - ASC (Ascending): növekvő sorrend (számoknál 0-9, szövegeknél A-Z).
    - DESC (Descending): csökkenő sorrend (számoknál 9-0, szövegeknél Z-A).
    - alapértelmezetten nővekvő sorrendben lesz rendezve ha nem adod meg a sorrendet

    SELECT * FROM tablanev ORDER BY mezonev [ASC | DESC]

    Rendezzük a termékeket név szerint.
    SELECT * FROM `termekek` ORDER BY `nev`;
    Rendezzük a termékeket kategória, majd név szerint.
    SELECT * FROM `termekek` ORDER BY `kategoria` DESC, `ar` DESC;

    Jelenítsünk meg 1 db terméket.

    Jelenítsük meg a legdrágább terméket.
    SELECT * FROM `termekek` ORDER BY `ar` DESC LIMIT 1;

    Jelenítsük meg a legdrágább élelmiszert.
    SELECT * FROM `termekek` WHERE `kategoria`='Élelmiszer' ORDER BY `ar` DESC LIMIT 1;

8. Aggregáló függvények
    - SUM(mező): Összeadja a megadott oszlop értékeit.
    - AVG(mező): Kiszámolja a megadott oszlop értékeinek átlagát.
    - MIN(mező): Megkeresi a legkisebb értéket.
    - MAX(mező): Megkeresi a legnagyobb értéket.
    - COUNT(mező vagy ): Megszámolja a sorok számát.
         A COUNT(*) az összes sort számolja, a COUNT(mező) csak azokat, ahol a mező értéke nem NULL.

9. Csoportosítás (GROUP BY)
    - A GROUP BY segítségével a sorokat csoportokba rendezhetjük egy vagy több oszlop értékei alapján
    - Aggregáló függvényekkel együtt használjuk általában

    - először csoportosítjuk az adatokat a megadot mezők szerint, majd a csoportokon belül elvégezzük a számítást

    SELECT kategoria, AVG(ar) FROM termekek GROUP BY kategoria;

10. Álnevek
    - Az álnevek segítségével ideiglenesen átnevezhetünk egy oszlopot vagy egy táblát a lekérdezés idejére.
    - aggregáló függvények eredményének adhatunk egy olvasható nevet ( AVG(ar) helyett atlagar )

    pl: SELECT nev AS 'Termék neve', ar AS 'Bruttó ár' FROM termekek;
    SELECT COUNT(*) AS 'Összes darabszám' FROM termekek;

11. Distinct

16. Táblák létrehozása és Záradékok (Constraints)
    A tábla létrehozásakor szabályokat (kényszereket) állíthatunk be az oszlopokra.
        - NULL / NOT NULL: Megengedjük-e, hogy az adott mező üresen maradjon?
        - UNIQUE: Az adott oszlopban nem szerepelhet két egyforma érték (pl. email cím).
        - PRIMARY KEY (Elsődleges kulcs): Egyedi azonosító, nem lehet NULL és nem ismétlődhet.
        - AUTO_INCREMENT: A szám magától növekszik minden új sornál (általában az ID-nál használjuk).
        - DEFAULT: Alapértelmezett érték, ha a felhasználó nem ad meg semmit.

    CREATE TABLE tablaneve (
        mezonev1 tipus zaradekok,
        mezonev2 tipus zaradekok,
    );
